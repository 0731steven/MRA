# 安全说明

## 报告问题

安全问题不应公开写入 Issue。交付后请通过企业指定的安全联系渠道报告，并附上受影响版本、复现条件和潜在影响。企业应在交付清单中补充实际联系人和响应时限。

## 部署要求

- 生产环境必须使用至少 32 个字符的随机 `SECRET_KEY` 和 PostgreSQL。
- `.env`、数据库备份、模型 API Key、教师邀请码和初始密码不得提交到 Git。
- 必须通过 HTTPS 对外提供服务，并限制数据库和管理端口的网络访问。
- `ALLOWED_ORIGINS` 只填写实际可信前端来源；同域部署可留空。
- 初始教师密码首次登录后应立即更换；企业负责账号停用和人员离职流程。
- 备份应加密、限制访问并定期执行恢复演练。

依赖漏洞、镜像扫描、日志审计、WAF、速率限制和隐私合规属于部署环境的持续安全工作，不应以一次代码交付代替。
